CodexClass CH5 signal map for the classroom touchpanel contract handoff.

**What:** This document defines the contract-ready signal names used by the CodexClass CH5-Svelte touchpanel. It covers the header online indicator, room power, master audio controls, and the independent six-source routing controls for both classroom displays. The names here match the UI bindings in the Svelte app and are intended to be imported into the later SIMPL or SIMPL# processor project. It does not include join numbers because those are allocated during Contract Editor build.

**Why:** The CH5 UI needs a stable namespace before the processor program is built so the front end and backend can stay aligned. Documenting the signals now reduces guesswork during manual SIMPL wiring and makes the UI behavior visible in SIMPL Debug once the contract is built and imported. This also gives a clean review surface for naming changes before join allocation happens. The document acts as the implementation contract for this repo.

**How:** Use the `CodexClass` symbol as the contract namespace in Contract Editor. Start from `contracts/CodexClass.cce`, review or rename placeholders there, then build the contract in Contract Editor so `.chd`, `.cse2j`, or `.g.cs` outputs are generated by the official toolchain. Import those generated files into SIMPL Windows or SIMPL# Pro rather than hand-authoring them. After any contract edit, rebuild and redeploy both the processor program and the CH5 archive.

## Header and Status

- `CodexClass.PanelOnline`
  Purpose: digital feedback shown in the header status pill.
  Default UI behavior: simulated `true` until backend feedback is wired.

## Room Power

- `CodexClass.RoomPowerToggle`
  Purpose: digital command pulse sent when the footer power control is pressed.
- `CodexClass.RoomPowerOn`
  Purpose: digital feedback for room power state.

## Master Audio

- `CodexClass.MasterVolumeUp`
  Purpose: digital pulse for master volume up.
- `CodexClass.MasterVolumeDown`
  Purpose: digital pulse for master volume down.
- `CodexClass.MasterMuteToggle`
  Purpose: digital pulse for mute toggle.
- `CodexClass.MasterMuted`
  Purpose: digital feedback for mute state.

## Display 1 Routing

- `CodexClass.Display1PcSelect`
- `CodexClass.Display1PcExtSelect`
- `CodexClass.Display1AppleTvSelect`
- `CodexClass.Display1AirMediaSelect`
- `CodexClass.Display1LaptopSelect`
- `CodexClass.Display1UsbCSelect`
- `CodexClass.Display1ActiveSource`
  Purpose: analog feedback where `1-6` maps to the active source above.
